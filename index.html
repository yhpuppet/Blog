Tips for Xcode

Posted on 2019-01-01 | In Xcode

Defines by Xcode

  • ${SRCROOT}: The path of ProjectName.xcodeproj.
  • ${PRODUCT_ROOT}

Breakpoints

Shared Breakpoint

Shared breakpoint is public to all project, so everyone in project will see it. Maybe if some coditions exist in project but you don’t want to execute in normal, use shared breakpoint to tell Xcode how to do. In combination with breakpoint options and automatically continuing breakpoints, this can also be very helpful to improve the general debugging experience.

User Breakpoint

User breakpoint is public to all project in your computer. It is useful to do some needed for all projects, e.g. set Exception Breakpoints, launch some debugging tools along with your app.

Symbol

  • UIViewAlertForUnsatisfiableConstraints
  • NSKVODeallocateBreak
  • UIApplicationMain
    Debugger command: e @import UIKit
    Imports UIKit into the debugger, removing the need to cast structs in a lot of places.
    e.g. p (CGRect)[self bounds]
    This will remove the need to cast to CGRect.
  • -[UIViewController initWithNibName:bundle:]
    Debugger command: po $arg1
    Prints the class of a view controller during its initialization.
  • -[UIApplication sendAction:toTarget:fromSender:forEvent:]
    Stops application when an event is delivered, such as the touch of a button. This can help figure out which message sent by the action.
    p (SEL)$arg3 will print the selector that is invoked.
    po $arg4 the target it is invoked on.
  • CGPostError
    Stops when CoreGraphics generates an error message, e.g. if you accidentally put a NaN in a drawing call somewhere.
  • _NSDataReinitializationBreak
    Stops when an already initialized NSData object gets initialized again.

Exception

  • Objective-C
    Debugger command: po $arg1
    Stops when an Objective-C breakpoint is hit, printing the exception to Xcode’s console.
  • C++
  • Swift
  • _XCTFailureHandler
    Breaks if a unit test produces an error. If you are running unit tests and want them to break if a test encounters an error, this is what you need.
  • Open GL ES

Xib and Nib

Posted on 2019-01-01

Nib

Nib是NeXT Interface Builder的缩写。

Xib

Xib是指XML Nib文件，于2007年Leopard(and Xcode 3.0)版本中更新。

相同点

Nib和Xib都是Interface Builder的图形界面设计文档。Interface Builder把窗口、菜单栏以及窗口上的各种控件的UI对象都“冻结”在了一个NIB文档里面；程序运行时，这些对象将会“苏醒”。

不同点

简单说，xib是nib文件的升级版。

  • Nib为二进制文件，Xib为纯文本文件
  • Xib方便于版本控制和Diff
  • Xib可编译为Nib。
